EduSpeak Scoring Packages

The table below provides a short description of the available EduSpeak scoring packages.  Each scoring package is designed to operate with a single master package from the "master_packages" directory.

The master packages that match these scoring packages, unless stated otherwise, are optimized for recognizing native speakers only.  Contact SRI if your application requires a master package simultaneously optimized for both recognition and scoring.

Please read this document carefully, including the notes at the bottom.  Some of the recognizer pronunciation scoring functions are supported only by a subset of the scoring packages.

To initialize scoring, set the post-initialization recognizer parameter client.ScoringConfigFile to the name of a scoring package.  To use any of these scoring packages, the recognizer rec.SegmentResults must be set to 1 on the recognizer command line.  Some scoring packages may require rec.SegmentResults to be set to 2 to utilize advanced pronunciation scoring features.

Here is an example command line that will create a recognizer utilizing scoring:

"-package C:/EduSpeak/sample-grammars/English/voice-mail rec.SegmentResults=1
-post client.ScoringDirectory=english-p16-na-991221.s991221"

Explanation of terms:

  • Scoring Package Name:  This is the name of an EduSpeak scoring package, which specifies the algorithm and parameters used for pronunciation scoring.  The scoring package must be used only with the master package named in column 2.
  • Master Package Name:  This is the name of an EduSpeak master package that should be used with the scoring package.
  • Language:  The language this master package is designed to recognize.
    • English:  North American English.
  • Age:  The speaker age range this scoring and master package pair is optimized to recognize.
  • Gender:  The speaker gender this scoring and master package pair is optimized to recognize.


IMPORTANT:  The following recognizer phone mispronunciation detection functions are currently only supported by the scoring package "english-p16-na-991221.s000608" and "english-p16-na-991221.s000608dc":

  • RCGetNumPhonesInSentence()
  • RCGetNumGraphemesInSentence()
  • RCGetBoundedPhoneScores()
  • RCGetBoundedPhoneScoresPhone()
  • RCGetBoundedPhoneScoresScore()
  • RCGetBoundedPhoneScoresMean()
  • RCScoringGetVectors()
  • RCScoringGetVectorsPhoneme()
  • RCScoringGetVectorsGrapheme()
  • RCScoringGetVectorsScore()


Notes:

  1. english-p16-na-991221.s000608:  This scoring package is identical to "english-p16-na-991221.s991221", except that it supports the recognizer mispronunciation detection functions listed above.  Rec.SegmentResults must be set to 2 when using the phone scoring functions.
  2. english-p16-na-991221.s000608dc:  This scoring package is identical to "english-p16-na-991221.s000608", except that it also supports duration and combination scoring. Rec.SegmentResults must be set to 2 when using these functions.
